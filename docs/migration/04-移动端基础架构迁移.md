# 步骤 04：移动端基础架构迁移

## 概述

将旧移动端 App 的基础架构迁移到新 Monorepo 中的 `apps/mobile/`，包括认证流程、导航结构、主题、Provider 层。

## 当前状态 vs 目标状态

| 维度 | 旧 App (pulsebuild-app) | 新 App (apps/mobile) |
|------|---------|---------|
| **Provider 层** | 手动嵌套 10+ Provider | 统一 Provider 组件 |
| **认证** | 自定义 SMS OTP + Zustand auth-store | Better Auth Client + Phone Plugin |
| **Token 存储** | expo-secure-store 手动管理 | Better Auth session 自动管理 |
| **导航** | Expo Router (auth)/(onboarding)/(app)/(tabs) | 保留结构，简化 |
| **主题** | React Native Paper MD3 自定义主题 | 迁移主题配置 |
| **API 客户端** | 手动 fetch + headers | tRPC + TanStack Query |
| **状态管理** | Zustand 17 stores | TanStack Query (服务端) + Zustand (客户端) |
| **错误边界** | 自定义 ErrorBoundary | 迁移复用 |

## 架构变更说明

### 1. 认证流程重构

**旧流程** (自定义实现):
```
手机号输入 → SMS API → 验证码验证 → JWT Token → SecureStore 存储
→ 每次请求手动附加 Bearer Token → 手动刷新 Token → 手动处理 401
```

**新流程** (Better Auth):
```
手机号输入 → Better Auth phoneNumber.sendOtp() → 验证码验证
→ Better Auth phoneNumber.verify() → Session 自动管理
→ tRPC 请求自动携带 Cookie/Session → 无需手动 Token 管理
```

### 2. 状态管理重构

**旧方案**: 17 个 Zustand store 各自管理服务端数据
```
project-store → 手动 fetch → cache → 手动 invalidate
task-store → 手动 fetch → cache → 手动 invalidate
...
```

**新方案**: TanStack Query 管理服务端状态
```
tRPC hooks (useQuery/useMutation) → 自动缓存 → 自动失效 → 自动重试
Zustand → 仅管理 UI 状态（filters, modals, theme preferences）
```

### 3. 需要保留的 Zustand Store

| Store | 保留原因 |
|-------|----------|
| `sync-status-store` | 网络/离线状态 (如果做离线支持) |
| `onboarding-store` | 引导流程 UI 状态 |

### 4. 迁移为 TanStack Query 的 Store

| 旧 Zustand Store | 新 tRPC Hook |
|-------------------|-------------|
| `project-store` | `trpc.project.list.useQuery()` |
| `task-store` | `trpc.task.list.useQuery()` |
| `subtask-store` | `trpc.subtask.list.useQuery()` |
| `defect-store` | `trpc.defect.list.useQuery()` |
| `site-diary-store` | `trpc.siteDiary.list.useQuery()` |
| `timesheet-store` | `trpc.timesheet.list.useQuery()` |
| `project-member-store` | `trpc.projectMember.list.useQuery()` |
| `assigned-work-store` | `trpc.task.listMyAssigned.useQuery()` |
| `connection-store` | 后期迁移 |
| `dependency-store` | 后期迁移 |

## 任务

### Phase 1: Better Auth 客户端配置

1. 安装依赖:
   ```
   pnpm --filter mobile add better-auth @better-auth/expo expo-secure-store
   ```

2. 创建 `src/lib/auth-client.ts`:
   - 使用 @better-auth/expo 的 expoClient 插件
   - 配置 baseURL 指向后端 API
   - 启用 phoneNumber 客户端插件
   - 配置 expo-secure-store 作为 token 存储
   - 导出 authClient 实例和类型化 hooks

### Phase 2: Provider 层重写

重写 `src/app/_layout.tsx`:
- 保留: PaperProvider, QueryClientProvider, TRPCProvider
- 新增: AuthProvider (Better Auth session 监听)
- 移除: 手动 token hydration 逻辑
- 添加: 根据认证状态自动导航到 (auth) 或 (app) 路由组
- 参考旧 _layout.tsx 的 Provider 嵌套顺序，但简化

### Phase 3: 认证页面

1. 创建 `src/app/(auth)/_layout.tsx` - Stack 导航
2. 创建 `src/app/(auth)/phone-entry.tsx`:
   - 迁移旧 app 的手机号输入 UI
   - 替换 API 调用为 authClient.phoneNumber.sendOtp()
   - 保留手机号 E164 格式化逻辑
3. 创建 `src/app/(auth)/code-verification.tsx`:
   - 迁移旧 app 的验证码输入 UI
   - 替换 API 调用为 authClient.phoneNumber.verify()
   - 验证成功后自动导航到 (app) 路由

### Phase 4: 已认证路由骨架

1. 创建 `src/app/(app)/_layout.tsx` - Tab 布局
   - 参考旧 app 的 tab 配置
   - 底部 Tab: 项目, 我的工作, 个人中心
2. 创建 `src/app/(app)/(tabs)/projects.tsx` - 空壳页面
3. 创建 `src/app/(app)/(tabs)/my-work.tsx` - 空壳页面
4. 创建 `src/app/(app)/(tabs)/profile.tsx` - 空壳页面

### Phase 5: 迁移工具函数

1. `src/lib/utils/phone-normalization.ts` - 从旧 app 迁移 E164 格式化
2. `src/hooks/use-auth.ts` - Better Auth session hook 封装
3. `src/hooks/use-network-status.ts` - 网络状态检测 (简化版)

### Phase 6: 主题迁移

1. `src/constants/theme.ts` - 迁移旧 app 的 MD3 主题定义
   - Tech Blue, Safety Orange, Cyber Cyan 配色
   - Light/Dark 主题
   - 只迁移颜色和核心主题，不迁移所有自定义 token
