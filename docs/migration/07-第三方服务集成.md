# 步骤 07：第三方服务集成

## 概述

旧项目深度绑定 Azure 生态 (Auth/Storage/Notifications)、SendGrid、Twilio 等。新项目需要重新选型。

## 服务清单与迁移建议

### 1. 文件存储

| 维度 | 旧方案 | 新方案选项 |
|------|--------|------------|
| 服务 | Azure Blob Storage | **选项 A: S3 兼容** (Cloudflare R2 / MinIO / AWS S3) |
| | | **选项 B: 保持 Azure Blob** |
| | | **选项 C: Supabase Storage** |
| SDK | @azure/storage-blob | aws-sdk (S3 兼容) 或 supabase-js |
| 费用 | Azure 定价 | R2 出流量免费，MinIO 自托管免费 |

### 2. 邮件服务

| 维度 | 旧方案 | 新方案选项 |
|------|--------|------------|
| 服务 | SendGrid | **选项 A: Resend** (开发者友好) |

| 用途 | 邮箱验证、密码重置、项目通知 | Better Auth 邮件 + 业务通知 |

### 3. 短信/OTP

| 维度 | 旧方案 | 新方案选项 |

### 4. 推送通知

| 维度 | 旧方案 | 新方案选项 |

### 5. AI 功能

| 维度 | 旧方案 | 新方案选项 |

### 6. 监控 & 分析

## 任务：集成第三方服务

### 1. 文件存储模块

创建 `src/modules/storage/`:
- `storage.module.ts` - 存储模块
- `storage.service.ts` - 抽象存储接口
- `storage.interface.ts`:
  ```typescript
  interface StorageProvider {
    upload(file: Buffer, key: string, contentType: string): Promise<{ url: string; key: string }>;
    download(key: string): Promise<Buffer>;
    delete(key: string): Promise<void>;
    getSignedUrl(key: string, expiresIn: number): Promise<string>;
  }
  ```
- `local-storage.provider.ts` - 本地文件系统实现（开发环境）
- `s3-storage.provider.ts` - S3 兼容实现（生产环境）
- 通过环境变量 STORAGE_PROVIDER 切换

### 2. 邮件模块

创建 `src/modules/email/`:
- `email.module.ts`
- `email.service.ts` - 邮件发送接口
- `email.interface.ts`:
  ```typescript
  interface EmailProvider {
    send(to: string, subject: string, html: string): Promise<void>;
    sendTemplate(to: string, templateId: string, data: Record<string, unknown>): Promise<void>;
  }
  ```
- `resend.provider.ts` - Resend 实现

### 3. SMS 集成到 Better Auth

更新 `src/modules/auth/auth.ts`:
- 在 phoneNumber plugin 的 sendOTP 回调中集成 SMS 发送
- 创建 `src/modules/sms/sms.service.ts`:
  ```typescript
  interface SmsProvider {
    send(to: string, message: string): Promise<void>;
  }
  ```
- 实现 Twilio provider 或 console.log fallback（开发环境）

### 4. 推送通知基础

创建 `src/modules/push/`:
- `push.module.ts`
- `push.service.ts` - 推送发送接口
- 暂时只实现接口，不接入具体服务
- 创建 device_registrations 表 schema

### 5. 环境变量

更新 `.env.example`:
```
# Storage
STORAGE_PROVIDER=local  # local | s3
S3_BUCKET=
S3_REGION=
S3_ACCESS_KEY_ID=
S3_SECRET_ACCESS_KEY=
S3_ENDPOINT=

# Email
RESEND_API_KEY=

# SMS
TWILIO_ACCOUNT_SID=
TWILIO_AUTH_TOKEN=
TWILIO_FROM_NUMBER=
```

## 规范要求
- 所有第三方服务使用接口抽象，方便更换
- 开发环境使用 mock/local 实现
- 生产环境通过环境变量配置
- 密钥从环境变量读取，绝不硬编码
- 每个服务独立模块，松耦合
```
