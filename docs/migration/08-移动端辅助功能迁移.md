# 步骤 08：移动端辅助功能迁移

## 概述

迁移旧 App 的辅助功能：国际化、监控、分析、无障碍访问、E2E 测试等。

## 功能清单

### 1. 国际化 (i18n)

| 维度 | 旧方案 | 迁移策略 |
|------|--------|----------|
| 框架 | i18next + react-i18next | 保持 |
| 语言 | 英文 + 中文 | 迁移翻译文件 |
| 配置 | lib/i18n/i18n.ts | 迁移到 src/lib/i18n/ |

### 2. 错误追踪

| 维度 | 旧方案 | 迁移策略 |
|------|--------|----------|
| 服务 | Sentry | 保持 |
| 配置 | lib/monitoring/sentry.ts | 迁移 |
| 导航集成 | 自动面包屑 | 适配 Expo Router |

### 3. 产品分析

| 维度 | 旧方案 | 迁移策略 |
|------|--------|----------|
| 服务 | PostHog | 可选迁移 |
| 事件 | 页面访问、功能使用 | 简化事件定义 |

### 4. 无障碍访问

| 维度 | 旧方案 | 迁移策略 |
|------|--------|----------|
| 标准 | WCAG 2.1 AA | 保持 |
| 实现 | accessibilityLabel, accessibilityRole | 迁移到组件中 |
| Hooks | use-reduced-motion.ts | 迁移 |

### 5. E2E 测试

| 维度 | 旧方案 | 迁移策略 |
|------|--------|----------|
| 框架 | Maestro | 评估是否保持 |
| 备选 | - | Detox / Appium |

### 6. 其他辅助功能

| 功能 | 旧实现 | 决策 |
|------|--------|------|
| 演示模式 | mock-auth-store + mock data | 后期 |
| 日历组件 | react-native-calendars | 按需迁移 |
| PDF 查看 | react-native-pdf | 按需迁移 |
| 底部弹窗 | @gorhom/bottom-sheet | 迁移 |
| Toast | 自定义 Provider | 迁移 |
| 实时翻译 | PostHog 驱动 | 不迁移 |

## Claude Code 提示词

```
你是 PulseBuild 项目的移动端开发者。当前项目路径: C:\projects\pulsebuild

前置条件：步骤 04-05 已完成。

参考旧项目:
- i18n: C:\projects\pulsebuild-projects\pulsebuild-app\lib\i18n\
- 常量/翻译: C:\projects\pulsebuild-projects\pulsebuild-app\constants\locales\
- Sentry: C:\projects\pulsebuild-projects\pulsebuild-app\lib\monitoring\sentry.ts
- 无障碍: C:\projects\pulsebuild-projects\pulsebuild-app\lib\utils\accessibility.ts

## 任务

### Phase 1: 国际化

1. 安装依赖:
   pnpm --filter mobile add i18next react-i18next expo-localization

2. 创建 `src/lib/i18n/i18n.ts`:
   - 从旧 app 迁移 i18next 配置
   - 使用 expo-localization 检测设备语言
   - fallback 语言: 英文

3. 迁移翻译文件到 `src/constants/locales/`:
   - en.json
   - zh.json
   - 只迁移当前已实现页面的翻译 key

4. 在 `_layout.tsx` 中初始化 i18n

### Phase 2: Sentry 错误追踪

1. 安装依赖:
   pnpm --filter mobile add @sentry/react-native

2. 创建 `src/lib/monitoring/sentry.ts`:
   - 从旧 app 迁移配置
   - DSN 从环境变量读取
   - 配置 Expo Router 导航集成

3. 在 `_layout.tsx` 中初始化 Sentry

### Phase 3: Bottom Sheet + Toast

1. 安装 @gorhom/bottom-sheet (如未安装):
   pnpm --filter mobile add @gorhom/bottom-sheet

2. 创建 `src/components/common/bottom-sheet.tsx`:
   - 封装 @gorhom/bottom-sheet
   - 统一 snap points 和样式

3. 创建 `src/components/common/toast-provider.tsx`:
   - 简单的 Toast 通知系统
   - 支持 success / error / info 类型

4. 添加到 `_layout.tsx` Provider 链

### Phase 4: 无障碍基础

1. 创建 `src/lib/utils/accessibility.ts`:
   - 迁移旧 app 的无障碍工具函数
   - accessibilityLabel 生成器
   - setAccessibilityFocus helper

2. 创建 `src/hooks/use-reduced-motion.ts`:
   - 检测用户是否启用了减少动效
   - 用于条件性禁用动画

## 不迁移:
- PostHog (后期按需)
- 演示模式 (mock auth)
- 实时翻译
- Maestro E2E 测试 (后期)
- PDF 查看器 (后期)
- 日历组件 (后期)

## 规范要求
- i18n key 使用点分命名: 'screen.projects.title'
- Sentry DSN 不硬编码
- 无障碍标签在每个可交互组件上
- Toast 自动消失时间 3 秒
```

## 需要你确认的事项

1. **i18n 语言**: 只支持英文+中文？还是需要更多语言？
2. **Sentry**: 是否沿用旧项目的 Sentry DSN？还是新建项目？
3. **PostHog**: 是否需要在本步骤集成？
4. **Bottom Sheet**: 使用 @gorhom/bottom-sheet 还是 React Native Paper 的 Modal？
5. **E2E 测试**: 保持 Maestro 还是切换到其他框架？
