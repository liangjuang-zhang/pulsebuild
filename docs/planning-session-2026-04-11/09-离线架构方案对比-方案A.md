# 离线优先架构方案 A: 双层数据架构

> Zustand + expo-sqlite + 自定义 SyncEngine

## 一、架构概览

```
┌────────────────────────────────────────────────────────────┐
│                    UI 层                                     │
│   TaskList / TaskEdit / SyncBadge                           │
└────────────────────────────────────────────────────────────┘
                           ↓
┌────────────────────────────────────────────────────────────┐
│              Zustand Store (统一状态层)                      │
│   taskStore: tasks[], editingTaskId, editingChanges        │
│   syncStatusStore: isOnline, pendingChanges, syncing       │
└────────────────────────────────────────────────────────────┘
                           ↓
┌────────────────────────────────────────────────────────────┐
│              Repository 层 (expo-sqlite 封装)                │
│   TaskRepository / SyncRepository / ProjectRepository       │
└────────────────────────────────────────────────────────────┘
                           ↓
┌────────────────────────────────────────────────────────────┐
│              SQLite (本地数据库)                             │
│   tasks 表 + sync_queue 表 + sync_metadata 表               │
└────────────────────────────────────────────────────────────┘
                           ↓ (网络恢复时)
┌────────────────────────────────────────────────────────────┐
│              SyncEngine (同步引擎)                           │
│   pushChanges() / pullChanges() / syncAll()                │
└────────────────────────────────────────────────────────────┘
                           ↓
┌────────────────────────────────────────────────────────────┐
│              tRPC API (服务端)                               │
│   task.create / task.update / sync.pull / sync.push        │
└────────────────────────────────────────────────────────────┘
```

```typescript
export const useTaskStore = create<TaskState>((set, get) => ({
  tasks: [],
  
  createTask: async (data: Partial<LocalTask>) => {
    // ─────────────────────────────────────────
    // 核心：Repository.create() 会自动：
    // 1. 写入 tasks 表（sync_status = 'pending')
    // 2. 写入 sync_queue 表（记录待同步操作）
    // ─────────────────────────────────────────

    const task = await repo.create({
      ...data,
      projectId: get().currentProjectId!,
    });
    
    // 更新本地状态（UI 立即显示）
    set((s) => ({ tasks: [...s.tasks, task] }));
    
    // 更新待同步计数
    useSyncStatusStore.getState().incrementPending();
    
    // 如果在线，立即尝试同步（后台执行，不阻塞 UI）
    const syncEngine = SyncEngine.getInstance();
    if (syncEngine.isOnline) {
      // 异步推送，用户可以继续操作
      syncEngine.pushChanges('task', task.id).catch(console.error);
    }
    
    return task;
  },
}));

export class TaskRepository {
  async create(data: Partial<LocalTask>): Promise<LocalTask> {
    const id = uuidv4();      // 本地 UUID
    const now = Date.now();
    
    // ─────────────────────────────────────────
    // SQLite 事务：同时写入任务表和同步队列
    // ─────────────────────────────────────────
    await db.transactionAsync(async (tx) => {
      // 1. 写入 tasks 表
      await tx.executeSqlAsync(
        `INSERT INTO tasks (
          id, project_id, name, status, priority,
          sync_status, local_created_at, local_updated_at
        ) VALUES (?, ?, ?, ?, ?, 'pending', ?, ?)`,
        [id, data.projectId, data.name, data.status, data.priority, now, now]
      );
      
      // 2. 写入 sync_queue 表（待同步队列）
      await tx.executeSqlAsync(
        `INSERT INTO sync_queue (
          entity_type, entity_id, operation, payload, created_at
        ) VALUES ('task', ?, 'create', ?, ?)`,
        [id, JSON.stringify(data), now]
      );
    });
    
    // 返回创建的任务
    return this.getById(id);
  }
}
```

```typescript
// apps/mobile/src/lib/database/repositories/task.repository.ts

export class TaskRepository {
  async update(id: string, changes: Partial<LocalTask>): Promise<void> {
    const now = Date.now();
    
    await db.transactionAsync(async (tx) => {
      // 1. 更新 tasks 表
      const fields = [];
      const values = [];
      
      if (changes.name) {
        fields.push('name = ?');
        values.push(changes.name);
      }
      if (changes.description) {
        fields.push('description = ?');
        values.push(changes.description);
      }
      if (changes.status) {
        fields.push('status = ?');
        values.push(changes.status);
      }
      
      // 标记为待同步
      fields.push('sync_status = ?', 'local_updated_at = ?');
      values.push('pending', now, id);
      
      await tx.executeSqlAsync(
        `UPDATE tasks SET ${fields.join(', ')} WHERE id = ?`,
        values
      );
      
      // 2. 写入 sync_queue
      // 注意：需要判断是 create 还是 update
      // 如果任务还没有 server_id，操作是 create（补充字段）
      // 如果任务已有 server_id，操作是 update
      const task = await this.getById(id);
      const operation = task?.serverId ? 'update' : 'create';
      
      await tx.executeSqlAsync(
        `INSERT INTO sync_queue (
          entity_type, entity_id, operation, payload, created_at
        ) VALUES ('task', ?, ?, ?, ?)`,
        [id, operation, JSON.stringify(changes), now]
      );
    });
  }
}

// SQLite 数据结果：
// tasks 表（更新后）:
// | id        | server_id | name     | sync_status | local_updated_at |
// |-----------|-----------|----------|-------------|------------------|
// | uuid-123  | NULL      | 已修改   | pending     | 1699001000       | ← 更新
//
// sync_queue 表（新增）:
// | id | entity_type | entity_id | operation | payload              | created_at |
// |----|-------------|-----------|-----------|----------------------|------------|
// | 1  | task        | uuid-123  | create    | {"name":"新任务"}    | 1699000000 | ← 旧的
// | 2  | task        | uuid-123  | create    | {"name":"已修改"}    | 1699001000 | ← 新的
```

### 场景 3: 网络恢复，自动同步

```
用户操作：无（自动触发）

网络状态：从离线 → 在线（走出地下室，恢复信号）

预期结果：
1. SyncEngine 自动检测网络恢复
2. 从 sync_queue 读取所有待同步操作
3. 推送到服务端 API
4. 更新本地 sync_status = 'synced'
5. 更新 server_id（如果是新创建的任务）
6. 清空 sync_queue
7. UI 更新徽章消失
```

#### 代码流程

```typescript
// 步骤 1: 网络监听
// apps/mobile/src/lib/sync/engine.ts

import NetInfo from '@react-native-community/netinfo';

export class SyncEngine {
  constructor() {
    // ─────────────────────────────────────────
    // 监听网络状态变化
    // ─────────────────────────────────────────
    NetInfo.addEventListener((state) => {
      const wasOffline = !this.isOnline;
      this.isOnline = state.isConnected ?? false;
      
      // 更新 Store 状态
      useSyncStatusStore.getState().setOnline(this.isOnline);
      
      // 网络恢复时，自动触发同步
      if (wasOffline && this.isOnline) {
        this.syncAll();
      }
    });
    
    // 定时同步（每 30 秒）
    setInterval(() => {
      if (this.isOnline) {
        this.syncAll();
      }
    }, 30000);
  }
  
  // ─────────────────────────────────────────
  // 推送变更：从 sync_queue 推送到服务端
  // ─────────────────────────────────────────
  async pushChanges(entityType: string): Promise<void> {
    const syncStatus = useSyncStatusStore.getState();
    syncStatus.startSync();
    
    try {
      // 获取待同步队列
      const queueResult = await db.executeSqlAsync(
        `SELECT * FROM sync_queue 
         WHERE entity_type = ? 
         ORDER BY created_at ASC`,
        [entityType]
      );
      
      const queueItems = queueResult.rows._array;
      
      for (const item of queueItems) {
        const payload = JSON.parse(item.payload);
        
        // ─────────────────────────────────────────
        // 根据 operation 调用不同 API
        // ─────────────────────────────────────────
        let serverResponse;
        
        if (item.operation === 'create') {
          // 创建操作：调用 task.create
          serverResponse = await trpc.task.create.mutate(payload);
          
        } else if (item.operation === 'update') {
          // 更新操作：需要先获取 server_id
          const repo = new TaskRepository();
          const local = await repo.getById(item.entity_id);
          
          if (local?.serverId) {
            serverResponse = await trpc.task.update.mutate({
              id: local.serverId,
              ...payload,
            });
          }
        }
        
        // ─────────────────────────────────────────
        // 更新本地状态
        // ─────────────────────────────────────────
        if (serverResponse) {
          const repo = new TaskRepository();
          
          // 标记为已同步，记录服务端 ID
          await repo.markSynced(
            item.entity_id,
            serverResponse.id,        // 服务端返回的 ID
            serverResponse.updatedAt  // 服务端时间戳
          );
          
          // 从队列删除
          await db.executeSqlAsync(
            `DELETE FROM sync_queue WHERE id = ?`,
            [item.id]
          );
          
          // 更新 Store 计数
          syncStatus.decrementPending();
        }
      }
      
      // 刷新 UI
      const projectId = useTaskStore.getState().currentProjectId;
      if (projectId) {
        useTaskStore.getState().loadTasks(projectId);
      }
      
    } catch (error) {
      // 记录错误，增加 retry_count
      await db.executeSqlAsync(
        `UPDATE sync_queue SET 
           retry_count = retry_count + 1,
           last_attempt_at = ?
         WHERE id = ?`,
        [Date.now(), item.id]
      );
      
      syncStatus.addError({ entityType, entityId: item.entity_id, error: error.message });
      
    } finally {
      syncStatus.finishSync();
    }
  }
  
  // ─────────────────────────────────────────
  // 拉取变更：从服务端获取最新数据
  // ─────────────────────────────────────────
  async pullChanges(entityType: string): Promise<void> {
    // 获取上次同步时间
    const metaResult = await db.executeSqlAsync(
      `SELECT last_sync_at FROM sync_metadata WHERE entity_type = ?`,
      [entityType]
    );
    
    const lastSyncAt = metaResult.rows._array[0]?.last_sync_at || 0;
    
    // 调用服务端 API，获取增量数据
    const serverData = await trpc.sync.pull.query({
      entityType,
      since: lastSyncAt,
    });
    
    // 更新本地数据库
    const repo = new TaskRepository();
    for (const serverTask of serverData.tasks) {
      await repo.upsertFromServer(serverTask);
    }
    
    // 更新同步元数据
    await db.executeSqlAsync(
      `INSERT OR REPLACE INTO sync_metadata (entity_type, last_sync_at)
       VALUES (?, ?)`,
      [entityType, Date.now()]
    );
    
    // 刷新 UI
    useTaskStore.getState().loadTasks(
      useTaskStore.getState().currentProjectId!
    );
  }
  
  // ─────────────────────────────────────────
  // 全量同步：推送 + 拉取
  // ─────────────────────────────────────────
  async syncAll(): Promise<void> {
    await this.pushChanges('task');
    await this.pushChanges('project');
    await this.pullChanges('task');
    await this.pullChanges('project');
  }
}

// 步骤 2: Repository 标记已同步
// apps/mobile/src/lib/database/repositories/task.repository.ts

export class TaskRepository {
  async markSynced(localId: string, serverId: string, serverUpdatedAt: number): Promise<void> {
    await db.executeSqlAsync(
      `UPDATE tasks SET 
         server_id = ?,           -- 记录服务端 ID
         sync_status = 'synced',  -- 标记已同步
         server_updated_at = ?    -- 记录服务端时间（用于冲突检测）
       WHERE id = ?`,
      [serverId, serverUpdatedAt, localId]
    );
  }
  
  // ─────────────────────────────────────────
  // 合并服务端数据（用于 pull）
  // ─────────────────────────────────────────
  async upsertFromServer(serverTask: ServerTask): Promise<void> {
    const now = Date.now();
    
    // 检查本地是否存在
    const existing = await db.executeSqlAsync(
      `SELECT * FROM tasks WHERE server_id = ?`,
      [serverTask.id]
    );
    
    if (existing.rows._array.length > 0) {
      const local = existing.rows._array[0];
      
      // ─────────────────────────────────────────
      // 冲突检测
      // ─────────────────────────────────────────
      if (local.sync_status === 'pending' && 
          local.local_updated_at > serverTask.updated_at) {
        // 本地有未同步的修改，且修改时间晚于服务端
        // → 冲突！标记为 conflict
        await db.executeSqlAsync(
          `UPDATE tasks SET 
             sync_status = 'conflict',
             server_updated_at = ?
           WHERE server_id = ?`,
          [serverTask.updated_at, serverTask.id]
        );
        
        // TODO: 通知用户处理冲突
        useSyncStatusStore.getState().addError({
          entityType: 'task',
          entityId: local.id,
          error: '数据冲突，需要手动处理',
        });
        
      } else {
        // 无冲突，直接更新本地数据
        await db.executeSqlAsync(
          `UPDATE tasks SET 
             name = ?, 
             status = ?, 
             description = ?,
             sync_status = 'synced',
             server_updated_at = ?, 
             local_updated_at = ?
           WHERE server_id = ?`,
          [serverTask.name, serverTask.status, serverTask.description,
           serverTask.updated_at, now, serverTask.id]
        );
      }
      
    } else {
      // 本地不存在，插入服务端数据
      await db.executeSqlAsync(
        `INSERT INTO tasks (
          id, server_id, project_id, name, status, description,
          sync_status, local_created_at, local_updated_at, server_updated_at
        ) VALUES (?, ?, ?, ?, ?, ?, 'synced', ?, ?, ?)`,
        [uuidv4(), serverTask.id, serverTask.project_id,
         serverTask.name, serverTask.status, serverTask.description,
         now, now, serverTask.updated_at]
      );
    }
  }
}

// SQLite 数据结果（同步后）:
// tasks 表:
// | id        | server_id | name     | sync_status | server_updated_at |
// |-----------|-----------|----------|-------------|-------------------|
// | uuid-123  | srv-456   | 已修改   | synced      | 1699001500        | ← 已同步
//
// sync_queue 表:
// | id | entity_type | entity_id | ... |  ← 已清空
// (空)
```

## 三、数据库 Schema 更新策略

### 问题：业务开发中如何处理数据库变更？

```
场景：业务需要给 tasks 表添加新字段 'due_date'

挑战：
1. 用户 APP 已安装旧版本（无 due_date 字段）
2. 新版本发布后，如何迁移旧数据？
3. 如何保证迁移不影响离线数据？
```

#### 方案：Drizzle-style Migration 系统

```typescript
        payload TEXT NOT NULL,

      )

      )

  },

// 002_add_due_date.ts - 添加截止日期字段
export const migration002 = {
  version: 2,
  up: async (db: SQLite.Database) => {
    // ─────────────────────────────────────────
    // SQLite 不支持 ALTER TABLE ADD COLUMN IF NOT EXISTS
    // 需要手动检查列是否存在
    // ─────────────────────────────────────────
    
    const columns = await db.executeSqlAsync(
      `PRAGMA table_info(tasks)`
    );
    
    const hasDueDate = columns.rows._array.some(
      (col: any) => col.name === 'due_date'
    );
    
    if (!hasDueDate) {
      await db.executeSqlAsync(`
        ALTER TABLE tasks ADD COLUMN due_date INTEGER NULL
      `);
      
      console.log('Migration 002: Added due_date column');
    }
  },
  
  down: async (db: SQLite.Database) => {
    // SQLite 不支持 DROP COLUMN，只能重建表
    await db.executeSqlAsync(`
      CREATE TABLE tasks_new AS 
      SELECT id, server_id, project_id, name, status, 
             sync_status, local_created_at, local_updated_at, server_updated_at
      FROM tasks
    `);
    
    await db.executeSqlAsync('DROP TABLE tasks');
    await db.executeSqlAsync('ALTER TABLE tasks_new RENAME TO tasks');
  },
};

export async function runMigrations(db: SQLite.Database): Promise<void> {
  // ─────────────────────────────────────────
  // 1. 确保 migrations 表存在
  // ─────────────────────────────────────────
  await db.executeSqlAsync(`
    CREATE TABLE IF NOT EXISTS migrations (
      version INTEGER PRIMARY KEY,
      applied_at INTEGER NOT NULL
    )
  `);
  
  // ─────────────────────────────────────────
  // 2. 获取已应用的 migration 版本
  // ─────────────────────────────────────────
  const result = await db.executeSqlAsync(
    `SELECT version FROM migrations ORDER BY version DESC LIMIT 1`
  );
  
  const currentVersion = result.rows._array[0]?.version || 0;
  
  console.log(`Current DB version: ${currentVersion}`);
  
  // ─────────────────────────────────────────
  // 3. 执行所有未应用的 migration
  // ─────────────────────────────────────────
  for (const migration of ALL_MIGRATIONS) {
    if (migration.version > currentVersion) {
      console.log(`Applying migration ${migration.version}...`);
      
      try {
        await migration.up(db);
        
        // 记录已应用
        await db.executeSqlAsync(
          `INSERT INTO migrations (version, applied_at) VALUES (?, ?)`,
          [migration.version, Date.now()]
        );
        
        console.log(`Migration ${migration.version} applied successfully`);
        
      } catch (error) {
        console.error(`Migration ${migration.version} failed:`, error);
        throw error;  // 抛出错误，APP 应提示用户
      }
    }
  }
  
  console.log('All migrations applied');
}
```

```typescript
// 问题：用户删除任务，如何同步到服务端？

// ─────────────────────────────────────────────────────
// 方案：软删除（sync_status = 'deleted') + Tombstone 表
// ─────────────────────────────────────────────────────

// apps/mobile/src/lib/database/repositories/task.repository.ts

export class TaskRepository {
  // ─────────────────────────────────────────
  // 删除任务：不直接 DELETE，而是标记状态
  // ─────────────────────────────────────────
  async delete(id: string): Promise<void> {
    const now = Date.now();
    
    await db.transactionAsync(async (tx) => {
      // 1. 标记任务为 deleted
      await tx.executeSqlAsync(
        `UPDATE tasks SET 
           sync_status = 'deleted',
           local_updated_at = ?
         WHERE id = ?`,
        [now, id]
      );
      
      // 2. 加入同步队列（operation = 'delete')
      await tx.executeSqlAsync(
        `INSERT INTO sync_queue (
          entity_type, entity_id, operation, payload, created_at
        ) VALUES ('task', ?, 'delete', ?, ?)`,
        [id, JSON.stringify({ deletedAt: now }), now]
      );
      
      // 3. 写入 tombstones 表（记录已删除的 ID，防止重复拉取）
      const task = await this.getById(id);
      if (task?.serverId) {
        await tx.executeSqlAsync(
          `INSERT OR IGNORE INTO sync_tombstones (
            entity_type, server_id, deleted_at
          ) VALUES ('task', ?, ?)`,
          [task.serverId, now]
        );
      }
    });
  }
}

async pullChanges(entityType: string): Promise<void> {
  const serverData = await trpc.sync.pull.query({
    entityType,
    since: lastSyncAt,
  });
  
  // 获取本地 tombstones
  const tombstones = await db.executeSqlAsync(
    `SELECT server_id FROM sync_tombstones WHERE entity_type = ?`,
    [entityType]
  );
  
  const deletedIds = new Set(
    tombstones.rows._array.map((t: any) => t.server_id)
  );
  
  // 过滤掉已删除的任务
  const filteredTasks = serverData.tasks.filter(
    (t: ServerTask) => !deletedIds.has(t.id)
  );
  
  // 更新本地数据库
  const repo = new TaskRepository();
  for (const serverTask of filteredTasks) {
    await repo.upsertFromServer(serverTask);
  }
}
```

## 四、数据冲突处理

### 冲突场景：用户离线修改 vs 服务端被他人修改

```
场景：
1. 用户 A 在离线状态下修改任务名称为 "已完成墙面"
2. 同时，用户 B 在服务端修改同一任务名称为 "需要返工"
3. 用户 A 网络恢复，尝试同步

冲突检测：
- 本地 local_updated_at = 1699000000
- 服务端 updated_at = 1699001000（晚于本地）
- 本地 sync_status = 'pending'（有未同步的修改）

→ 冲突！
```

#### 冲突处理策略

```typescript
// apps/mobile/src/lib/sync/conflict-resolver.ts

export enum ConflictStrategy {
  SERVER_WINS = 'server',      // 服务端优先（默认）
  CLIENT_WINS = 'client',      // 客户端优先
  MANUAL = 'manual',           // 让用户选择
  MERGE = 'merge',             // 字段合并
}

export class ConflictResolver {
  // ─────────────────────────────────────────
  // 检测冲突
  // ─────────────────────────────────────────
  detectConflict(local: LocalTask, server: ServerTask): boolean {
    // 本地有未同步的修改
    if (local.syncStatus !== 'pending') {
      return false;  // 无冲突
    }
    
    // 服务端更新时间晚于本地最后同步时间
    if (server.updated_at > (local.serverUpdatedAt || 0)) {
      return true;  // 有冲突
    }
    
    return false;
  }
  
  // ─────────────────────────────────────────
  // 解决冲突（默认策略：服务端优先）
  // ─────────────────────────────────────────
  resolve(
    local: LocalTask, 
    server: ServerTask,
    strategy: ConflictStrategy = ConflictStrategy.SERVER_WINS
  ): LocalTask {
    
    if (strategy === ConflictStrategy.SERVER_WINS) {
      // 服务端数据覆盖本地
      return {
        ...local,
        name: server.name,
        status: server.status,
        description: server.description,
        syncStatus: 'synced',
        serverUpdatedAt: server.updated_at,
      };
      
    } else if (strategy === ConflictStrategy.CLIENT_WINS) {
      // 客户端数据优先，但需要重新推送
      return {
        ...local,
        syncStatus: 'pending',  // 保持 pending，等待推送
      };
      
    } else if (strategy === ConflictStrategy.MANUAL) {
      // 标记为冲突，等待用户处理
      return {
        ...local,
        syncStatus: 'conflict',
        conflictData: {
          localName: local.name,
          serverName: server.name,
          localUpdatedAt: local.localUpdatedAt,
          serverUpdatedAt: server.updated_at,
        },
      };
      
    } else if (strategy === ConflictStrategy.MERGE) {
      // 字段合并：保留本地修改的字段，其他字段用服务端数据
      const localChanges = this.getChangedFields(local);
      
      return {
        ...local,
        // 未修改的字段用服务端数据
        ...server,
        // 修改的字段保留本地数据
        ...localChanges,
        syncStatus: 'pending',  // 合并结果需要推送
      };
    }
  }
  
  // ─────────────────────────────────────────
  // 获取本地修改的字段
  // ─────────────────────────────────────────
  private getChangedFields(local: LocalTask): Partial<LocalTask> {
    // 从 sync_queue 获取修改记录
    const queueResult = await db.executeSqlAsync(
      `SELECT payload FROM sync_queue 
       WHERE entity_id = ? AND operation = 'update'
       ORDER BY created_at DESC LIMIT 1`,
      [local.id]
    );
    
    if (queueResult.rows._array.length > 0) {
      return JSON.parse(queueResult.rows._array[0].payload);
    }
    
    return {};
  }
}

// ─────────────────────────────────────────
// UI 冲突处理页面
// ─────────────────────────────────────────

// apps/mobile/src/screens/ConflictResolver.tsx

function ConflictResolverScreen({ taskId }: { taskId: string }) {
  const { tasks } = useTaskStore();
  const task = tasks.find(t => t.id === taskId);
  
  if (!task || task.syncStatus !== 'conflict') {
    return null;
  }
  
  const { conflictData } = task;
  
  const handleResolve = async (choice: 'local' | 'server' | 'merge') => {
    const resolver = new ConflictResolver();
    
    // 从服务端获取最新数据
    const serverTask = await trpc.task.detail.query({ id: task.serverId! });
    
    // 根据用户选择解决冲突
    const strategy = choice === 'local' 
      ? ConflictStrategy.CLIENT_WINS 
      : choice === 'server' 
        ? ConflictStrategy.SERVER_WINS 
        : ConflictStrategy.MERGE;
    
    const resolved = resolver.resolve(task, serverTask, strategy);
    
    // 更新本地数据
    const repo = new TaskRepository();
    if (choice === 'server') {
      await repo.update(task.id, resolved);
    } else {
      // 本地或合并：标记为 pending，等待推送
      await repo.update(task.id, resolved);
      
      // 立即推送
      SyncEngine.getInstance().pushChanges('task', task.id);
    }
    
    // 返回列表页
    router.back();
  };
  
  return (
    <View>
      <Text>⚠️ 数据冲突</Text>
      
      <Text>本地修改 ({formatTime(conflictData.localUpdatedAt)}):</Text>
      <Text>任务名称: {conflictData.localName}</Text>
      
      <Text>服务端数据 ({formatTime(conflictData.serverUpdatedAt)}):</Text>
      <Text>任务名称: {conflictData.serverName}</Text>
      
      <Button onPress={() => handleResolve('local')} title="使用本地版本" />
      <Button onPress={() => handleResolve('server')} title="使用服务端版本" />
      <Button onPress={() => handleResolve('merge')} title="合并两者" />
    </View>
  );
}
```

## 五、总结：方案 A 特点

### 优点

1. **迁移成本低**：源项目 Repository 逻辑可直接复用
2. **概念清晰**：SQLite → Repository → Store → UI，经典三层架构
3. **完全控制**：精确控制同步逻辑、冲突策略
4. **调试友好**：直接查看 SQLite 数据，问题定位快
5. **离线优先**：所有操作先写本地，网络恢复后同步

### 缺点

1. **手动维护**：sync_status、sync_queue 需手动管理
2. **同步逻辑复杂**：pushChanges/pullChanges 需手动实现
3. **冲突检测**：需要手动比较时间戳
4. **Migration 管理**：需要手动编写和执行 migration

### 适用场景

- 需要精确控制同步逻辑
- 团队熟悉 SQL 和 Repository 模式
- 迁移源项目代码（复用 Repository）
- 需要自定义冲突策略
